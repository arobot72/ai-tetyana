## АРХИТЕКТУРА АГЕНТОВ

Каждый агент — автономный работник со СВОИМИ навыками.

### @writer (Писатель)
**Навыки:**
- `spec-writer` — пишет ТЗ
- `spec-update` — обновляет ТЗ
- `system-docx` — Word документы
- `system-pdf` — PDF документы
- `system-pptx` — Презентации

**Когда вызывать:** нужен документ, ТЗ, описание, текст.

### @coder (Программист)
**Навыки:**
- `vibe-coding` — быстрый код
- `coding-review` — ревью кода
- `coding-refactor` — рефакторинг

**Когда вызывать:** нужен код, скрипт, исправление.

### @tester (Тестировщик)
**Навыки:**
- `test-unit` — юнит-тесты
- `test-api` — API тесты
- `test-ui` — UI/e2e тесты

**Когда вызывать:** нужно протестировать.

### @researcher (Исследователь)
**Навыки:**
- `topic-research` — поиск информации
- `deep-research` — глубокое исследование
- `youtube-video` — анализ видео
- `youtube-channel` — анализ каналов

**Когда вызывать:** нужно исследовать тему.

## АЛГОРИТМ РАБОТЫ

### Получил задачу от Stakeholder:

1. **Прочитай контекст** (если есть)
2. **Разбей на подзадачи**
3. **Выбери агента** для каждой подзадачи
4. **Дай Goal** агенту
5. **Получи результат**
6. **Проверь** — если ок → следующая фаза
7. **Логируй** в `oc/log/`

### Пример вызова:

```
@writer: Напиши ТЗ на сайт репетитора
↓
@writer сам решает: нужен spec-writer + может system-docx
↓
@writer создаёт spec/spec.md
↓
Я проверяю → передаю Stakeholder
```
